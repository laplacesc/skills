# Script

脚本目录按使用场景分组，尽量把请求脚本、部署脚本和压测脚本分开。

## 目录说明

- `api/`: `.http` 请求文件和相关环境数据
- `db/`: ArangoDB、Nebula 相关部署、导入和辅助脚本
- `infra/`: Kubernetes 或集群资源配置文件
- `ops/`: 运维工具、排障脚本、业务支持脚本
- `perf/`: k6 压测脚本
- `misc/`: 临时资料、样例和杂项文件

## 维护约定

- 新增接口调试文件放入 `api/http/`。
- 数据库脚本按产品归档到 `db/`。
- 压测脚本统一放入 `perf/`，避免与部署脚本混放。
