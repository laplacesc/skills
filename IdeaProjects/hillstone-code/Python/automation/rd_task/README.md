# RD 任务排期助手

登录 hillstone RD 系统，获取 FR 任务列表，基于中国节假日自动计算排期，批量写回系统。

## 项目结构

```
rd_task/
├── rd_task_core.py      # 核心模块：WorkdayCalculator + RDClient（无 UI 依赖）
├── rd_task_app.py       # Flet Web 版 (v4.1, 需要 flet>=0.85)
└── requirements.txt     # Flet 版依赖
```

## 架构

```
┌─────────────────────────────────────┐
│         rd_task_core.py             │
│  WorkdayCalculator    RDClient      │
│  (节假日计算)          (RD API)      │
└──────────────────┬──────────────────┘
                   │
            ┌──────┴────────┐
            │  Flet Web/桌面 │
            └───────────────┘
```

`rd_task_core.py` 不依赖任何 GUI 框架。

## 运行

```bash
pip install -r requirements.txt

# Web 模式（浏览器打开）
python rd_task_app.py --web

# 桌面模式
python rd_task_app.py
```

## 功能特性

- 登录 RD 系统并自动获取指定 FR 下属于当前用户的任务
- 任务列表支持内联编辑工作量、其他工作量、起止日期
- 日期字段同时支持文本直接输入和日历组件选择
- 自动计算排期（跳过周末和中国法定节假日）
- 相邻任务可勾选"合并"共享同一起止日期
- 批量将工时和排期写回 RD 系统
- 浏览器自动填充用户名密码（autofill hints）
- 操作日志实时滚动显示

## 数据流

```
登录 RD → 获取 FR 任务树 → 提取当前用户的叶子任务
    → 按日期分组计算 other_workdays
    → 用户编辑工作量 / 勾选合并 / 选择起始日期
    → 计算排期（跳过节假日和周末）
    → 确认后批量 POST 回 RD 系统
    → 自动重新拉取同步数据
```

## 排期计算逻辑

- 0.1 人天至少消耗 1 个工作日跨度
- 1.5 人天消耗 2 个工作日跨度
- 通过 [NateScarlet/holiday-cn](https://github.com/NateScarlet/holiday-cn) 获取中国官方节假日
- 连续勾选"合并"的任务归为一组，同组共享相同的起止日期
